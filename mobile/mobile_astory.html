CASE 04 / AI NARRATIVE MOBILE EXPERIENCE

ASTORY

一个围绕债务生存展开的 AI 叙事经营系统

Astory 是一款 Android 移动游戏原型。玩家在债务压力、路线风险与有限资源中做选择，通过跑商交易、地图探索、NPC 交涉、角色关系和轮回成长推进主线目标。

02 / PRODUCT POSITION

不是单纯的文字冒险，而是状态驱动的叙事经营产品。

定位AI 辅助叙事 + 资源管理
用户喜欢策略、剧情、角色关系和经营成长的移动端玩家
重点让复杂系统始终被玩家看懂

03 / CORE LOOP

玩家每天都在权衡：赚钱、恢复、冒险，还是推进关系？

04 / SYSTEM ARCHITECTURE

用五个主入口，把复杂系统压缩成清晰的用户心智。

  • 旅途系统：控制风险、体力和路径选择。
  • 地图系统：让探索、收益和事件连接起来。
  • 市场系统：用动态交易制造经营压力。
  • 人物系统：承载关系、信任和 AI 交涉。
  • 轮回系统：把失败转化为下一次成长。

05 / AI NEGOTIATION

AI 不是聊天入口，而是进入玩法闭环的叙事层。

NPC 交涉谈判、请求、打探、交易
技能玩家能做什么
传闻他人怎么看你
情感结晶关系系统的长期回报

06 / NATIVE BUILD

从 UX 结构，到 Android Studio 原生实现。

项目不只停留在概念，而是将信息架构、页面状态、流程跳转与原生界面组织到 Android 实现里。

07 / REFLECTION

复杂系统的关键，是让玩家始终知道下一步能做什么。

这个项目的收获，是把叙事、经营、关系和 AI 对话压缩成移动端可理解的结构。不是让系统显得复杂，而是让复杂始终有清楚入口。
